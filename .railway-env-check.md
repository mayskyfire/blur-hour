# Railway Environment Variables Check

## ปัญหา: Environment variables ไม่ถูกโหลด

จาก console log: `hasApiKey: false, hasAuthDomain: false, hasProjectId: false`

## วิธีแก้ไข:

### 1. ตรวจสอบใน Railway Dashboard

ไปที่: **Railway Dashboard > Your Project > Variables**

ต้องมีตัวแปรทั้งหมดนี้:

```
NUXT_PUBLIC_FIREBASE_API_KEY=AIzaSyCsD29ycdQ22Vs1k5MEuyY850wjrLvTRbw
NUXT_PUBLIC_FIREBASE_AUTH_DOMAIN=blur-hour.firebaseapp.com
NUXT_PUBLIC_FIREBASE_PROJECT_ID=blur-hour
NUXT_PUBLIC_FIREBASE_STORAGE_BUCKET=blur-hour.firebasestorage.com
NUXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=812200063590
NUXT_PUBLIC_FIREBASE_APP_ID=1:812200063590:web:16e67cb708fd020d9ef3a7
```

### 2. ตรวจสอบว่าใส่ถูกต้อง

- ✅ ชื่อตัวแปรต้องมี `NUXT_PUBLIC_` prefix
- ✅ ไม่มีช่องว่างหน้าหรือหลัง
- ✅ ไม่มี quotes (" หรือ ')
- ✅ กด Save/Apply หลังจากเพิ่ม

### 3. Redeploy

หลังจากเพิ่ม/แก้ไข variables:
- คลิก **Redeploy** ใน Railway
- หรือ push commit ใหม่

### 4. ตรวจสอบ Build Logs

ใน Railway Dashboard > Deployments > Latest > Build Logs

ดูว่ามี error หรือไม่

### 5. ตรวจสอบ Deploy Logs

ใน Railway Dashboard > Deployments > Latest > Deploy Logs

ดูว่า environment variables ถูกโหลดหรือไม่
